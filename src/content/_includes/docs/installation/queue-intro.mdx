<p>ThingsBoard uses a message queue to route messages between its internal services. Select the option that matches your infrastructure:</p>
<ul>
  <li><strong>In Memory</strong> (default) — built-in queue, no extra setup required. Suitable for development and PoC. Not recommended for production or multi-node deployments.</li>
  <li><strong>Kafka</strong> — high-throughput, durable queue. Run it yourself or use a managed service such as <a href="https://aws.amazon.com/msk/" target="_blank">AWS MSK</a>.</li>
  <li><strong>Confluent Cloud</strong> — fully managed Kafka service. Use this if you want Kafka without managing the infrastructure yourself.</li>
</ul>
