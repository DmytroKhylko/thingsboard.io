ThingsBoard PE <a href="/docs/pe/user-guide/white-labeling-general/">white-labeling</a> allows you to configure your company or product logo and color scheme.
In addition to the web interface, white-label settings also apply to the ThingsBoard PE Mobile app.
You can customize the login screen and the main app interface.

## Login screen white-labeling

As a System or Tenant Administrator, configure the login screen logo and color palette in the **Login White Labeling** form.

<Aside type="note">
By default, system-level login white-labeling is applied (configured by the System Administrator). Each tenant can override these settings by specifying a **Domain name** in the **Login White Labeling** form.
Tenant-level settings are applied based on the domain name of the API endpoint configured in the mobile app (the `thingsBoardApiEndpoint` constant in `lib/constants/app_constants.dart`).
</Aside>

<Steps>

1. Go to **White Labeling** → **Login White Labeling** through the main menu.
2. Enter your domain in the **Domain name** field for tenant-level settings.
3. Enter the base URL in the **Base URL** field (includes protocol — `http` or `https` — and port if it differs from the standard `80` or `443`).
4. Upload your logo image in the **Logo** field.
5. Specify the logo height if necessary in the **Logo height** field.
6. Select a primary color palette in the **Primary palette** field.
7. Click **Save** to apply changes.

</Steps>

## Main application interface white-labeling

As a System or Tenant Administrator, configure the main app interface logo and color palette in the **White Labeling** form.

<Aside type="note">
By default, system-level white-labeling is applied (configured by the System Administrator). Each tenant can override these settings in the **White Labeling** form.
</Aside>

<Steps>

1. Go to **White Labeling** → **White Labeling** through the main menu.
2. Upload your logo image in the **Logo** field.
3. Select a primary color palette in the **Primary palette** field.
4. Click **Save** to apply changes.

</Steps>
