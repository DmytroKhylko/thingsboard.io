**Alarms** are notifications generated when a monitored condition is met, like a temperature exceeding a threshold or a device going offline. ThingsBoard creates, updates, and clears alarms automatically based on the logic defined in <DocLink product={props.product} path="user-guide/alarm-rules">alarm rules</DocLink>.

## Alarm properties

Every alarm contains the following fields:

- **Originator:** The entity that triggered the alarm, like a Device or Asset.
- **Type:** A string identifying the alarm category, like *High Temperature*. ThingsBoard uses the type to match new events to an existing active alarm rather than creating duplicates.
- **Severity:** One of five levels indicating how critical the condition is. See [Alarm severity](#alarm-severity).
- **Status:** The current lifecycle state of the alarm. See [Alarm states](#alarm-states).
- **Start time:** Timestamp when the alarm condition was first met.
- **End time:** Timestamp when the alarm condition was last met.

## Alarm states

An alarm moves through states based on the condition status and user actions.

| State | Description |
| :--- | :--- |
| **Active Unacknowledged** | The condition is active and no user has acknowledged the alarm. |
| **Active Acknowledged** | The condition is active and a user has acknowledged the alarm. |
| **Cleared Unacknowledged** | The condition is no longer active, but the alarm has not been acknowledged. |
| **Cleared Acknowledged** | The condition is no longer active and the alarm has been acknowledged. |

## Alarm severity

Severity indicates how critical the alarm condition is and helps prioritize response.

| Level | Meaning |
| :--- | :--- |
| **Critical** | Requires immediate action for conditions like equipment failure, safety risks, or potential data loss. |
| **Major** | Significant issue that affects operation but is not immediately critical. |
| **Minor** | A less severe condition to monitor or address later. |
| **Warning** | A potential issue that does not yet affect normal operation. |
| **Indeterminate** | Severity cannot be determined, often due to incomplete or ambiguous data. |

Severity is set per trigger condition in the alarm rule. When an existing alarm is active and a new trigger fires at a higher severity, ThingsBoard upgrades the severity of the existing alarm rather than creating a new one.

## Alarm propagation

By default, an alarm is visible only at the entity that triggered it. Propagation extends that visibility up the entity hierarchy, like making an alarm from a Device visible on the Building Asset that contains it.

Propagation is configured per alarm rule. See the <DocLink product={props.product} path="user-guide/alarm-rules">Alarm rules</DocLink> page for the available propagation options.

## Manage alarms

### View alarms

To view all alarms visible to your account, go to **Alarms** in the left-hand menu.

To view alarms for a specific entity:

<Steps>
1. Go to the entity list, like **Entities → Devices**.
2. Click the entity name to open its details.
3. Select the **Alarms** tab.
</Steps>

### Acknowledge and clear

<Aside type="note">
  Clearing an alarm does not delete it from the database. It remains available for historical reporting until explicitly deleted.
</Aside>

You can acknowledge and clear alarms manually through the UI or automatically via alarm rules.

- **Acknowledge:** Marks the alarm as seen. Records the Ack time and the user who acknowledged it. The alarm condition may still be active.
- **Clear:** Marks the alarm condition as resolved. Records the Clear time.

To acknowledge or clear an alarm, open the **Alarms** page or the entity's **Alarms** tab, find the alarm row, and click **Acknowledge** or **Clear**.


## Alarm widgets

Alarms can be displayed on dashboards using widgets from the **Alarm widgets** bundle. The most commonly used is the **Alarms table**, which lists alarms for selected entities and supports filtering by status, severity, and time range. Other widgets in the bundle include alarm counts, alarm duration charts, and individual alarm details.

To add an alarm widget, open a dashboard, click **Add widget**, and search for *Alarm* under **Alarm widgets**.

For a full walkthrough of building and configuring dashboards, see <DocLink product={props.product} path="user-guide/dashboards">Dashboards</DocLink>.
