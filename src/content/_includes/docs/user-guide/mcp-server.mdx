The [ThingsBoard MCP Server](https://github.com/thingsboard/thingsboard-mcp) lets you query platform data, generate sample telemetry, and detect anomalies using **natural language** via an MCP-enabled LLM client (e.g., Claude Desktop).

With ThingsBoard MCP you can:

- **Find devices** matching specific criteria in your ThingsBoard platform.
- **Simulate or ingest telemetry** (e.g., AQI sensor data) directly into ThingsBoard for a specific device.
- **Analyze device data** for anomalies — easily adapted to search for spikes, data gaps, or other patterns.

## Prerequisites

Before you begin, ensure that you have:

- A ThingsBoard account — either on [ThingsBoard Cloud](https://thingsboard.cloud/signup) or a self-hosted installation.
- Authentication credentials — either an <DocLink product={props.product} path="user-guide/security/api-keys">API key</DocLink> (recommended, requires ThingsBoard 4.3+) or username and password.
- An **MCP-enabled LLM client** — for example, [Claude Desktop](https://claude.ai/download), [Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview), VS Code with an MCP extension, or any other client that supports the [Model Context Protocol](https://modelcontextprotocol.io/).

## Step 1. Connect MCP server to ThingsBoard

Start your LLM Agent with the ThingsBoard MCP Server.
This guide uses [Claude Desktop](https://claude.ai/download), but any MCP-enabled client works.

**Pull the Docker image:**

```bash
docker pull thingsboard/mcp
```

**Configure your MCP client** — the examples below use Claude Desktop (`claude_desktop_config.json`), but any MCP-enabled client works with the same Docker command and environment variables.

<Tabs>
<TabItem label="API Key (recommended)">
```json maxLines=20 collapsible
{
  "mcpServers": {
    "thingsboard": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "THINGSBOARD_URL",
        "-e",
        "THINGSBOARD_API_KEY",
        "-e",
        "LOGGING_PATTERN_CONSOLE",
        "thingsboard/mcp"
      ],
      "env": {
        "THINGSBOARD_URL": "<thingsboard_url>",
        "THINGSBOARD_API_KEY": "<your_api_key>",
        "LOGGING_PATTERN_CONSOLE": ""
      }
    }
  }
}
```

Replace `<thingsboard_url>` with your ThingsBoard host URL and `<your_api_key>` with your <DocLink product={props.product} path="user-guide/security/api-keys">API key</DocLink>.

<Aside type="note">
  API key authentication requires ThingsBoard **4.3+**. The MCP server inherits the permissions of the user the API key was created for — use a dedicated user with appropriate permissions for your use case.
</Aside>
</TabItem>
<TabItem label="Username & Password">
```json maxLines=20 collapsible
{
  "mcpServers": {
    "thingsboard": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "THINGSBOARD_URL",
        "-e",
        "THINGSBOARD_USERNAME",
        "-e",
        "THINGSBOARD_PASSWORD",
        "-e",
        "LOGGING_PATTERN_CONSOLE",
        "thingsboard/mcp"
      ],
      "env": {
        "THINGSBOARD_URL": "<thingsboard_url>",
        "THINGSBOARD_USERNAME": "<thingsboard_username>",
        "THINGSBOARD_PASSWORD": "<thingsboard_password>",
        "LOGGING_PATTERN_CONSOLE": ""
      }
    }
  }
}
```

Replace `<thingsboard_url>`, `<thingsboard_username>`, and `<thingsboard_password>` with your actual values.
</TabItem>
</Tabs>

## Real-world use cases

The ThingsBoard MCP Server extends beyond AQI anomaly detection. It applies to a wide range of scenarios:

| Use case | Description |
|---|---|
| **Industrial IoT** | Detect unusual vibration, temperature, or pressure changes in machinery. |
| **Energy management** | Identify consumption spikes in smart meters and optimize energy usage. |
| **Environmental monitoring** | Detect sudden changes in air or water quality and trigger alerts. |
| **Predictive maintenance** | Analyze equipment performance trends and anticipate failures. |
| **Smart city** | Monitor traffic flow, street lighting, or waste collection anomalies. |

By combining MCP's natural language interface with ThingsBoard's data visualization and alerting capabilities, you can rapidly prototype and deploy AI-driven IoT solutions across many industries.
