The Visualization Layer transforms raw telemetry into interactive control panels. Dashboards aggregate multiple data sources into a single pane of glass for real-time monitoring and remote hardware control.

- **[Dashboard](/docs/user-guide/dashboards/):** Centralized UI container for visualizing data and managing entities. Supports multi-page layouts and role-based access control.
- **[Widget](/docs/user-guide/widgets/):** Modular UI element (chart, gauge, map, or table) used to display telemetry or execute commands.
- **[Alias](/docs/user-guide/ui/aliases/):** Dynamic reference mapping a widget to a specific device or asset. Allows a single dashboard to show data for different entities without redesigning the UI.
- **[Dashboard State](/docs/user-guide/dashboards/#dashboard-states):** Logical "page" or view within a dashboard. Used to build drill-down hierarchies (e.g., clicking a "Building" asset opens a state showing only its "Room" sensors).
- **[Widget Bundle](/docs/user-guide/ui/widget-library/):** Curated collection of widgets grouped by function, such as *Digital Gauges*, *Charts*, or *Control Widgets*.
