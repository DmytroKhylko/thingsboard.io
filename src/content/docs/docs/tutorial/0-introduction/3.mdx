[Alarm](/docs/{{docsPrefix}}user-guide//alarms/) in ThingsBoard represents an abnormal condition or critical event related to an entity, such as a device. Alarms are generated automatically when conditions defined in [alarm rules](/docs/{{docsPrefix}}user-guide/alarm-rules/) are met (for example, when a telemetry value exceeds a threshold).

Alarm notifications are delivered through the ThingsBoard [Notification center](/docs/{{docsPrefix}}user-guide/notifications/), which allows you to notify users about alarms and other system events via email, SMS, push notifications, or third-party integrations.

:::tip[Configure email notifications for alarms]
To learn how to configure email notifications for alarms, see the [Send email on alarm](/docs/tutorials/send-email/) guide.
:::

Use the [ThingsBoard Mobile Application](/docs/mobile/) to enable push notifications that instantly deliver alerts about critical events in your IoT ecosystem directly to your smartphone—wherever you are.

In this step, you will configure an alarm rule that detects when the temperature exceeds a defined threshold.  
Then, you will simulate this condition by publishing telemetry data above the threshold to trigger the alarm and generate a notification.

---

## Step 3.1 Configure alarm rule

Create an alarm rule that triggers when the temperature exceeds 25 °C.

:::tip
Configure the alarm rule at the **device profile** level to ensure the same logic is applied to all devices using this profile.
:::

#### Add the alarm rule

<Steps>
1. Navigate to **Alarms &#8702; Alarm rules**. 

2. Click **"&#43; Add alarm rule"** in the upper-right corner and select **"Create new alarm rule"**.
</Steps>

In the alarm rule configuration dialog, specify the following:

##### General settings

<Steps>
1. Alarm type: **High temperature**

2. Target entity: **default**
</Steps>

##### Add a telemetry argument

<Steps>
1. Click **Add argument**

2. Configure the argument:   
   &#8226;&#8194;Entity type: **Current entity**   
   &#8226;&#8194;Argument type: **Latest telemetry**   
   &#8226;&#8194;Time series key: `temperature`   
   &#8226;&#8194;Argument name: `temperature`

3. Click **Add**
</Steps>
  
This creates the <span class="code-light">temperature</span> variable used in the alarm condition.

##### Define the trigger condition

<Steps>
1. Click **Add trigger condition**.

2. Ensure that **Severity** is set to **Critical**.

3. **Add a condition** in **Script** mode:
   ```javascript
   // Triggers an alarm when temperature is above 25 degrees
   return temperature > 25;
   ```
4. Click **Save**.
</Steps>

##### Save the rule

Click **Add** to save and activate the alarm rule.

The alarm will now be triggered whenever the device temperature exceeds **25 °C**.

---

## Step 3.2 Trigger an Alarm

Publish telemetry with a value greater than **25 °C** (for example, `26`).   
Use the Check connectivity feature as described in [Step 1.2](http://localhost:4324/docs/tutorial/0-introduction/1/#step-12-verify-device-connectivity) to send the test telemetry data.

Once the threshold is exceeded, the new active alarm can be viewed, acknowledged, or cleared in the following places:
- on the **Alarms** page;
- on the dashboard using the **Alarms widget**;
- the **Alarms** tab in the device details page.

You can also view the new alarm notification by clicking the **bell icon** in the top-right corner of the screen or by opening the **Notification center**.

This demonstrates how ThingsBoard detects and reacts to critical conditions in real time.
