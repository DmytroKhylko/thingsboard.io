ThingsBoard supports multi-tenancy, allowing you to securely share devices and dashboards with different customers while keeping their data fully isolated.

[Customer](/docs/{{docsPrefix}}user-guide/ui/customers/) in ThingsBoard is a logical container used to isolate data and provide controlled access for external users under a tenant.  

**Customer User** is a user account linked to a specific customer. Customer users have read-only access and can view only the entities assigned to their customer.

In this step, you will:
- create a Customer and a Customer User
- assign a device and dashboard to the Customer
- verify that the Customer User can access only the assigned data

This demonstrates how ThingsBoard ensures secure data isolation between customers.

---

## Step 4.1 Create a Customer and a Customer User

Create a new customer:

<Steps>
1. Navigate to the **Customers**.

2. Click the **&#43; Add customer**.

3. Enter **My New Customer** as the title.   
Optionally, provide an address, phone number, or other additional details.

4. Click **Add**.
</Steps>

The customer has been created. Now, create a user for this customer:

<Steps>
1. In the **Customers** list, click **Manage customer users** for the newly created customer.

2. Click **&#43; Add user** in the top-right corner.

3. Enter the user&#39;s email address.  
  Optionally, specify the first name, last name, phone number, preferred UI language, and unit system.

4. Click **Add**.

5. **Copy the activation link** and store it in a secure place — you will use it later to activate the new user account.

6. Click **OK** to finish.
</Steps>
  
A Customer User account has been created.

---

## Step 4.2 Assign the Device to the Customer

To allow the Customer User to interact with the device and view its telemetry, assign the device to the Customer.

<Steps>
1. In the **Customers** list, click **Manage customer devices** for **My New Customer**.

2. Click the **&#43; Assign new device** in the top-right corner of the table.

3. Select **Thermometer** from the list and click **Assign**.
</Steps>
  
The device is assigned to the Customer.

---

## Step 4.3 Assign the Dashboard to the Customer

Now assign the dashboard to the customer to allow their users to view the data displayed on it. Customer users will have **read-only** access to the dashboard and will not be able to add, modify, or remove widgets.

<Steps>
1. In the **Customers** list, click **Manage customer dashboards** for **My New Customer**.

2. Click the **&#43; Assign new dashboard** in the top-right corner of the table.

3. Select **My Dashboard** from the list and click **Assign**.
</Steps>

Customer users now have access to the dashboard.

:::tip[Optionally]

You can configure a dashboard to be displayed immediately after the user logs in to the ThingsBoard web UI.

<Steps>
1. In the **Customers** list, click **Manage customer users** for **My New Customer**.

2. Click the user to open its details.

3. Click the orange **pencil** icon to enter edit mode.

4. Set **My Dashboard** as the **default dashboard**.

5. Enable **Always fullscreen**.

6. Apply changes.
</Steps>
:::
